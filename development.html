More project documentation

In project documentation

The project sources contain much development documentation in various documentation files. In particular:

  • featurespecs/ contains documentations for the various features.
  • Architecture contains a description of the chosen project architecture and records architectural decisions.
  • Project structure describes the module structure of the project.
  • ChatGPT API Analysis contains a discussion of the ChatGPT API wrt. our project.
  • Next steps contains a sketch of the next steps to be taken.

The Composum AI issues contains more concrete plans for enhancing it, and is a place where you are welcome to place suggestions and bug reports!

Development with ChatGPT and Github Copilot

Part of the intention of the project is to evaluate the use of AI services, in particular ChatGPT and the Github Copilot IntelliJ plugin, for speeding up development. Some outcomes of that:

  • The [feature creation process](https://github. com/ist-dresden/composum-AI/tree/develop/featurespecs/FeatureCreationProcess.md) massively improves the quality and speed of specifying a feature and thinking it through. See the feature specification directory for examples of that.
  • Speeding up the development is ongoing, but Github Copilot and ChatGPT certainly speed up a lot of things.
